git 入门

git vs svn

  • git 是分布式，svn 是集中式
  • git 是一个指针，svn 的分支是文件复制
  • git 保存文件快照，svn 保存文件差异

三种基本状态

git 入门

snapshot, commit, branch

  • 保存被修改文件的完整快照，而不是差异
  • 每个commit指向若干个snapshot
  • 除了root，每个commit至少有一个父节点
  • 分支就是一个指向commit的指针

git 入门

snapshot, commit, branch

git 入门

fetch vs pull

  • 三个概念：远程分支，本地仓库中的远程分支拷贝，本地分支
  • fetch: 远程分支复制到本地仓库中
  • merge: 合并两个分支
  • pull: pull == merge + fetch

git 入门

merge vs rebase

  • merge：合并两个分支，产生一个新的 commit
  • rebase：把一个分支的修改在另一个分支上重做一遍，不会产生新的 commit
  • 如果两个分支是 fastforward 关系，merge 等价于 rebase

git 入门

branch vs tag

  • 相同点：都是指向一个 commit 的指针
  • 不同点：
    • branch: 一系列 commit 的序列
    • tag: 某一个特定的状态

git 入门

reset vs revert

  • reset: 直接移动当前分支的head指针，把之后的提交退回到stage中，或者加 --hard 参数直接丢弃
  • revert：新建一个 commit，这个 commit 的作用是撤销指定的commit中的修改

git 入门

submodule

  • 作用：在一个 repo 中包含另一个 repo
  • 什么时候用？：当前 repo 中的另一个 repo 比较大，或者想定期更新其中的代码，或者只在某种特定情况下才会用到这个repo

git 入门

新建分支 vs fork 仓库

  • 少数人合作的小任务，可以共用一个仓库，各自创建分支开发，定期合并代码
  • 多人合作的大项目（比如nodejs这种开源项目），为了避免分支太多，每个人应该自己fork一个仓库，开发完成后向主仓库发 pull request。同时应该关闭其他人的push权限，限制只能用pull request
